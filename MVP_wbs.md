## 1. 專案總覽 (Project Overview)

本計劃聚焦於 6 週內交付 MVP 核心價值。透過「基礎先行、功能並行」的開發節奏，確保影音流轉管道的技術可行性與穩定性。

- **五大核心任務**：
    1. 環境架設與資料庫建立 (基礎)
    2. 用戶註冊/登入 (Google SSO)
    3. 課程瀏覽/詳情
    4. 課程建立/上傳影片 (後台)
    5. 影片播放 (核心)

---

## 2. WBS 結構總覽 (WBS Structure Overview)

1. 1.0 環境架設與資料庫 (Foundational Setup)
2. 2.0 用戶認證系統 (User Auth - Google SSO)
3. 3.0 課程內容展示 (Course Browsing & Detail)
4. 4.0 內容管理與上傳 (Content Ingestion)
5. 5.0 影片串流播放 (Playback Experience)

## 4. 專案進度摘要 (Project Progress Summary)

### 📈 關鍵路徑里程碑

1. **M1 (Week 1)**: **環境穩定化**。所有團隊成員可進行代碼同步，資料庫可正常寫入。
2. **M2 (Week 2)**: **認證閉環**。用戶可透過 Google 帳號正常進出平台。
3. **M3 (Week 4)**: **上傳-播放打通**。影片上傳後可成功轉碼並在前端播放。
4. **M4 (Week 6)**: **交付版本**。完成 Bug 修復，正式部署至生產環境。

## 5. 風險管理 (Risk Management)

| **風險項目** | **類別** | **影響程度** | **緩解措施** |
| --- | --- | --- | --- |
| **DB Schema 變更頻繁** | 基礎 | 高 | 第一週進行強制性 Schema 評審，後續變更需透過 Migration 工具管理。 |
| **影片轉碼資源限制** | 技術 | 中 | 使用雲端託管轉碼服務 (GCP Transcoder API) 以應對高併發。 |
| **Google SSO 配置錯誤** | 外部 | 中 | 開發首日即建立 GCP Console 專案並配置 Redirect URI。 |

## 6. 品質指標與里程碑 (Quality Metrics)

- **後端效能**：所有唯讀 API 響應時間 < 400ms (95th percentile)。
- **前端品質**：首屏渲染 (LCP) < 2.5s。
- **資料一致性**：影片上傳狀態必須與資料庫同步，不允許出現「死連結」。

**專案管理總結**: 本 WBS 確保了從「基礎環境 (1.0)」到「功能實現 (2.0-5.0)」的完整鏈條。特別是將資料庫設計放在首位，是為了避免後續開發中因數據模型不一致導致的大規模重構。
